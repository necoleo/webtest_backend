【配置端】【管理端】【小程序】工作流节点操作人支持申请人角色功能测试用例
测试用例编号	测试用例名称	优先级	前置条件	操作步骤	预期结果
01	配置端支持工作流节点设置处理人为"申请人"	P0	配置端配置流程模板	1.新建流程模板;2.添加除系统节点外的所有节点;3.点击节点配置，设置操作人，检查是否可选择申请人选项;4.检查申请人的角色字段	1.可以设置申请人作为操作人;2比较大的roleValue字段为applicant
02	创建流程批次实例支持不设置操作人	P0	项目专员创建批次实例	1.申请人节点点击选择操作人;2.检查弹出的操作人设置界面，不设置申请人;3.点击取消;4.发布批次实例	1.操作人设置界面里，弹窗的title为申请人设置;2.设置申请人可见受助人范围默认为结对受助人，没有所有受助人选项;3.有申请人加入任务的操作提示;4.显示现有申请人列表;5.创建实例允许不设置操作人
03	创建流程批次实例支持设置操作人	P0	项目专员创建批次实例	1.创建流程批次实例时设置具体操作人;2.操作人列表所显示的申请人为已提交申请过受助人的申请人	1.创建批次实例允许设置具体操作人;2.操作人列表所显示的申请人为已提交申请过受助人的申请人
04	管理端_申请人可见受助人范围校验	P0	已有包含申请人操作人的流程批次实例	1.添加申请任务中未立项的受助人加入立项流程任务;2.添加绑定过受助人的申请人（所绑定的受助人已加入任务）;3.添加未绑定受助人的申请人	1.流程批次添加受助人实例成功;2.只有该申请人绑定的受助人的节点操作人为该申请人，其他受助人在申请人节点没有操作人;3.未绑定受助人的申请人，无法成为受助人实例申请人操作节点的操作 hemodynamics
05	小程序_申请人可见受助人范围校验	P0	已有包含申请人操作人的流程批次实例	1.查看任务列表;2.查看批次实例	1.设置申请人后，小程序能看到立项流程的任务;2.只能看到申请人所绑定的受助人的任务;3.无法查看其他未绑定、绑定了其他申请人的受助人
06	允许小程序用户扫码作为申请人加入任务	P0	已有包含申请人操作人的流程批次实例	1.小程序用户扫码;2.点击加入任务;3.查看任务列表	1.点击加入任务时，若还不是申请人，则需要先注册申请人才能加入任务;2.若已是申请人，则直接加入任务;3.任务列表里包含该任务
07	节点通用操作	P1	已有实例到申请人节点	1.节点有关联表单则提交表单;2.节点没有关联表单则点击通过;3.点击撤回该节点;4.点击驳回该节点;5.点击终止该节点	1.提交表单成功，管理端可以看到所提交的表单信息;2.通过成功，管理端可以看到提交 CCR状态;3.撤回节点成功，可以再次提交;4.驳回该节点成功，前节点操作完后可继续操作;5.终止该节点成功，流程结束
