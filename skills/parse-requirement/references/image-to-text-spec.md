# 图文融合规范

## 必须描述的元素

### 1. 页面标识
- 页面标题
- 面包屑导航
- URL路径

### 2. 布局结构
- 整体分区（头部/内容区/底部，左侧栏/主区域）
- 各区域的大致比例

### 3. 交互元素

| 元素类型 | 必须描述的内容 |
|---------|--------------|
| 按钮 | 文字内容、位置、样式（主要/次要/危险/禁用） |
| 输入框 | 标签文字、占位符文字、是否必填（*标记） |
| 下拉框 | 标签、当前选中值 |
| 表格 | 所有列名、示例数据行 |
| 标签页 | 各标签名称、当前激活项 |

### 4. 数据内容
- 表格中的示例数据
- 列表项的具体内容
- 表单的默认值

### 5. 状态标识
- 禁用/启用状态
- 选中/未选中状态
- 展开/收起状态
- 错误/警告提示

## 输出格式

在原型图位置插入描述，使用以下格式：

```
【原型图开始：{页面名称}】
{逐层描述图中内容}
【原型图结束】
```

## 描述顺序

1. 先整体后局部：先描述页面整体布局，再逐区域细化
2. 从上到下：按照视觉阅读顺序描述
3. 从左到右：同一行内按从左到右顺序

## 描述示例

```
【原型图开始：用户列表页】

页面标题：用户管理
面包屑：首页 > 系统设置 > 用户管理

布局：
- 顶部：页面标题栏
- 左侧：无侧边栏
- 主区域：搜索栏 + 数据表格

搜索栏（页面顶部）：
- 输入框：用户名（占位符：请输入用户名）
- 输入框：手机号（占位符：请输入手机号）
- 按钮：查询（主要样式，蓝色）
- 按钮：重置（次要样式，白色）
- 按钮：新增用户（主要样式，蓝色，位于右侧）

数据表格：
- 列名：序号、用户名、手机号、角色、状态、创建时间、操作
- 示例数据行：
  - 1 | admin | 138****1234 | 管理员 | 启用 | 2024-01-01 | 编辑 删除
  - 2 | user1 | 139****5678 | 普通用户 | 禁用 | 2024-01-02 | 编辑 删除
- 操作列：编辑（文字链接）、删除（文字链接，红色）

分页：
- 显示：共 100 条，每页 10 条
- 页码：1 2 3 ... 10

【原型图结束】
```

## 禁止事项

- ❌ 描述图中不存在的元素
- ❌ 推测功能的实现方式
- ❌ 添加原图没有的交互说明
- ❌ 改变原文档的任何文字内容
- ❌ 对视觉设计做主观评价（如"设计简洁"）
